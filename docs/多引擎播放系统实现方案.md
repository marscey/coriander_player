# 多引擎播放系统实现方案

## 一、项目背景与需求分析

Coriander Player 是一款支持 Windows、macOS、Android 及 iOS 的跨平台本地与私有云音乐播放器。目前播放器基于 BASS 音频库构建，在播放在线音频（流式播放）方面存在不足。本方案旨在引入 media_kit 库构建新的播放引擎，同时保留 BASS 库的实现方案，支持通过配置设置切换不同的播放引擎。

### 核心需求
- 引入 media_kit 库实现流式播放功能
- 保留现有的 BASS 库实现
- 支持通过配置切换不同的播放引擎
- 默认策略：桌面端（Windows、macOS）采用 BASS 库，移动端（Android、iOS）采用 media_kit 库

## 二、技术选型分析

### 1. BASS 库现状
- **优势**：高性能的音频播放，支持多种音频格式解码，提供强大的DSP效果
- **不足**：流式播放支持有限，移动端适配性不佳，授权成本较高
- **适用场景**：本地音乐播放，桌面平台

### 2. media_kit 库分析
- **优势**：专为 Flutter 设计的跨平台媒体播放库，对流式播放支持良好，移动端集成度高
- **不足**：相比 BASS 的DSP处理能力较弱，对某些无损音频格式支持有限
- **适用场景**：在线音乐播放，移动平台

## 三、架构设计方案

### 1. 整体架构

采用抽象工厂模式和策略模式，实现播放引擎的解耦和动态切换：

```
┌─────────────────────────────────────────────┐
│              播放服务层 (PlaybackService)     │
└───────────────┬─────────────────────────────┘
                │

```

### 2. 核心组件设计

#### 2.1 播放器引擎接口 (PlayerEngine)

定义统一的播放器引擎接口，规定所有播放器引擎必须实现的方法：

```dart
abstract class PlayerEngine {
  // 初始化播放器
  Future<void> initialize();
  
  // 设置音频源
  Future<void> setSource(String path, {bool isAsset = false, bool isNetwork = false});
  
  // 播放控制
  Future<void> play();
  Future<void> pause();
  Future<void> stop();
  Future<void> seek(Duration position);
  
  // 音频属性控制
  void setVolume(double volume);
  void setSpeed(double speed);
  
  // 状态获取
  PlayerState get state;
  Duration get position;
  Duration get duration;
  
  // 状态流
  Stream<PlayerState> get playerStateStream;
  Stream<Duration> get positionStream;
  
  // 释放资源
  Future<void> dispose();
}
```

#### 2.2 引擎工厂 (PlayerEngineFactory)

负责创建并返回适当的播放器引擎实例：

```dart
class PlayerEngineFactory {
  static PlayerEngine createEngine(PlayerEngineType type) {
    switch (type) {
      case PlayerEngineType.bass:
        return BassPlayerEngine();
      case PlayerEngineType.mediaKit:
          return MediaKitPlayerEngine();
      default:
        throw UnimplementedError('Unsupported player engine type');
    }
  }
  
  // 根据平台和配置获取默认引擎
  static PlayerEngine getDefaultEngine() {
    final config = AppSettings.instance;
    final engineType = config.playerEngineType;
    
    if (engineType != null) {
      return createEngine(engineType);
    }
    
    // 根据平台选择默认引擎
    switch (defaultTargetPlatform) {
      case TargetPlatform.android:
      case TargetPlatform.iOS:
        return createEngine(PlayerEngineType.mediaKit);
      case TargetPlatform.windows:
      case TargetPlatform.macOS:
      default:
        return createEngine(PlayerEngineType.bass);
    }
  }
}
```

### 3. 播放服务适配

修改 `PlaybackService` 类，使其使用 `PlayerEngine` 接口而非直接依赖 `BassPlayer`：

```dart
class PlaybackService {
  late PlayerEngine _playerEngine;
  
  PlaybackService() {
    _initialize();
  }
  
  Future<void> _initialize() async {
    // 获取默认引擎
    _playerEngine = PlayerEngineFactory.getDefaultEngine();
    await _playerEngine.initialize();
    
    // 监听状态变化
    _setupStateListeners();
  }
  
  // 切换引擎方法
  Future<void> switchEngine(PlayerEngineType type) async {
    // 保存当前播放状态
    final currentPosition = _playerEngine.position;
    final currentAudio = currentAudio;
    
    // 释放旧引擎资源
    await _playerEngine.dispose();
    
    // 创建新引擎
    _playerEngine = PlayerEngineFactory.createEngine(type);
    await _playerEngine.initialize();
    _setupStateListeners();
    
    // 恢复播放状态
    if (currentAudio != null) {
      await setAudio(currentAudio);
      _playerEngine.seek(currentPosition);
    }
    
    // 更新配置
    AppSettings.instance.playerEngineType = type;
  }
  
  // 其他方法适配为使用_playerEngine
}
```

### 4. 配置管理

在 `AppSettings` 中添加播放器引擎类型配置：

```dart
class AppSettings {
  PlayerEngineType? _playerEngineType;
  
  PlayerEngineType? get playerEngineType => _playerEngineType;
  
  set playerEngineType(PlayerEngineType? type) {
    _playerEngineType = type;
    _saveSettings();
  }
  
  // 从存储加载配置
  void _loadSettings() {
    // 现有加载逻辑
    
    final engineTypeStr = prefs.getString('playerEngineType');
    if (engineTypeStr != null) {
      _playerEngineType = PlayerEngineType.values.byName(engineTypeStr);
    }
  }
  
  // 保存配置到存储
  void _saveSettings() {
    // 现有保存逻辑
    
    if (_playerEngineType != null) {
      prefs.setString('playerEngineType', _playerEngineType!.name);
    } else {
      prefs.remove('playerEngineType');
    }
  }
}
```

### 5. 设置页面扩展

在设置页面添加引擎选择功能：

```dart
class EngineSelectionSetting extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final settings = AppSettings.instance;
    
    return SettingsTile(
      title: '播放器引擎',
      subtitle: _getEngineDisplayName(settings.playerEngineType),
      onTap: () async {
        final result = await showDialog<PlayerEngineType>(
          context: context, 
          builder: (context) => SimpleDialog(
            title: Text('选择播放器引擎'),
            children: [
              SimpleDialogOption(
                child: Text('BASS 引擎 (推荐桌面端)'),
                onPressed: () => Navigator.pop(context, PlayerEngineType.bass),
              ),
              SimpleDialogOption(
                child: Text('media_kit 引擎 (推荐移动端)'),
                onPressed: () => Navigator.pop(context, PlayerEngineType.mediaKit),
              ),
            ],
          ),
        );
        
        if (result != null && result != settings.playerEngineType) {
          // 显示加载提示
          showDialog(
            context: context, 
            barrierDismissible: false,
            builder: (context) => AlertDialog(
              content: Row(
                children: [
                  CircularProgressIndicator(),
                  SizedBox(width: 16),
                  Text('正在切换播放器引擎...'),
                ],
              ),
            ),
          );
          
          // 切换引擎
          await PlayService.instance.playbackService.switchEngine(result);
          
          // 关闭加载提示
          Navigator.pop(context);
          
          // 显示成功提示
          ScaffoldMessenger.of(context).showSnackBar(
            SnackBar(content: Text('播放器引擎已切换')),
          );
        }
      },
    );
  }
  
  String _getEngineDisplayName(PlayerEngineType? type) {
    if (type == null) {
      return '根据平台自动选择';
    }
    return type == PlayerEngineType.bass ? 'BASS 引擎' : 'media_kit 引擎';
  }
}
```

## 六、测试与迁移计划

### 1. 测试计划

- **单元测试**：验证 `PlayerEngine` 接口的所有方法是否正确实现
- **集成测试**：测试引擎切换功能是否正常工作，状态保存和恢复是否正确
- **性能测试**：比较两种引擎在不同平台上的播放性能、内存占用和电池消耗
- **兼容性测试**：确保所有支持的音频格式都能在两种引擎下正常播放

### 2. 迁移路线图

1. **第一阶段**：完成 `PlayerEngine` 接口定义和 BASS 引擎适配器实现
2. **第二阶段**：实现 media_kit 引擎适配器
3. **第三阶段**：修改 `PlaybackService` 以使用新的引擎抽象层
4. **第四阶段**：添加配置管理和引擎切换功能
5. **第五阶段**：进行全面测试和性能优化
6. **第六阶段**：更新文档和用户指南

## 七、风险评估与应对

### 1. 潜在风险

- **性能风险**：新的引擎实现可能在某些场景下性能不如现有实现
- **兼容性风险**：某些音频格式或功能可能在新引擎下不被支持
- **稳定性风险**：引入新库可能带来新的 bugs 和稳定性问题

### 2. 应对策略

- **性能监控**：添加性能监控代码，收集两种引擎的性能数据
- **兼容性测试**：创建全面的兼容性测试套件
- **回滚机制**：确保用户可以随时切换回原来的引擎
- **渐进式迁移**：默认保留原有行为，让用户选择是否切换到新引擎

## 八、总结

本方案通过引入抽象工厂模式和策略模式，实现了播放器引擎的解耦和动态切换。这不仅解决了当前播放引擎在流式播放方面的不足，还为将来可能的引擎扩展打下了良好的基础。通过合理的跨平台适配策略和详细的测试计划，确保了新系统的稳定性和兼容性。
